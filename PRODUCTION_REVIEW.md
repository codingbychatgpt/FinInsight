# FinInsight 后续开发与上线清单

更新时间：2026-06-11

## 项目定位

FinInsight 是少量受邀用户使用的私有金融信息工具，预计用户不超过 10 人。

- 不提供游客模式和公开注册，未登录用户只能进入登录页。
- 权限分为 `user` 和 `admin`，两类角色功能严格隔离。
- `user` 使用业务站点：阅读新闻、同步今日热点、网络搜索、URL 导入、AI 解析、AI 对话、收藏和阅读记录。
- `admin` 只使用管理后台：管理用户、新闻源、文章内容、导入记录、失败日志、备份和系统状态。
- `admin` 不能访问普通用户业务功能，`user` 不能访问管理功能；服务端必须校验角色，不能只依赖前端隐藏入口。
- 不需要围绕大规模访问或爬虫资源消耗设计复杂限制，但同步仍保留互斥，避免并发写入重复数据。

## 核心开发范围

### 1. 登录与角色隔离

- 增加登录页、登录、退出和当前用户接口。
- 账号由管理员创建，不开放注册。
- 密码使用 Argon2 或 bcrypt 哈希保存。
- 建议使用服务端 Session 或 `HttpOnly + Secure` Cookie。
- `user` 登录后进入业务主页，`admin` 登录后直接进入管理后台。
- 登录体系完成后删除同步接口 Basic Auth。

### 2. 网络搜索与 URL 导入

网络搜索和 URL 导入应与自动同步并列，导入后的新闻继续使用现有详情页、AI 解析和 AI 助手流程。

URL 导入流程：

1. 用户粘贴新闻或政策 URL。
2. 系统规范化 URL 并检查数据库是否已存在。
3. 抓取标题、来源、发布日期和正文。
4. 展示预览，允许用户修正字段。
5. 确认后写入数据库，状态设为 `pending`。
6. 用户可以立即执行 AI 解析。

网络搜索流程：

1. 用户输入关键词。
2. 系统通过可配置搜索服务返回候选网页。
3. 用户选择结果后进入统一 URL 导入流程。

要求：

- 手动导入历史新闻不受自动同步“两天日期范围”限制。
- URL 导入复用现有 URL 唯一索引、正文抓取和 AI 解析能力。
- 增加 `ingestion_method`、`imported_by`、`imported_at` 等导入元数据。
- 防范 SSRF：禁止 localhost、内网 IP、云元数据地址、非法协议，并校验每次重定向。

### 3. 将“来源权重”改为“来源分布”

- 当前面板展示的是各来源文章数量，并不是真实权重。
- 面板改名为“来源分布”，展示全库来源文章数和占比。
- 点击来源可以直接筛选政策流。
- 未来如果增加可靠性和抓取成功率评分，再单独增加“来源质量”面板。

### 4. 搜索、筛选和排序

- 按标题、正文和 AI 关键词搜索。
- 按来源、风险等级、解析状态、发布日期范围和导入方式筛选。
- 按最新发布、风险分数和最近导入时间排序。
- 支持分页或加载更多，不再只查看最新 15 条。
- 点击来源分布、风险分层和关键词热区时直接应用筛选条件。

### 5. 收藏与阅读记录

- 收藏、取消收藏新闻。
- 进入详情页后自动记录已读状态和最近阅读时间。
- 主页支持只看未读和只看收藏。
- 增加“我的收藏”和“最近阅读”视图。
- 收藏和阅读记录与 `user` 账号绑定。

### 6. 详情页增强

- 收藏、已读状态和复制链接。
- 根据关键词、来源和发布时间推荐相关政策。
- 展示同关键词历史政策时间线。
- AI 助手快捷问题和用户对话历史。
- 复制或导出 AI 解读。
- 显示文章来自自动同步、网络搜索导入或 URL 手动导入。

### 7. 管理后台

管理员只进入独立管理后台，首版包含：

- 创建、禁用用户并设置 `user/admin` 角色。
- 查看和管理新闻源。
- 查看同步、网络搜索和 URL 导入记录。
- 删除、隐藏或重新解析文章。
- 查看抓取失败和 AI 解析失败原因。
- 查看服务健康、数据库备份状态和 AI 调用统计。
